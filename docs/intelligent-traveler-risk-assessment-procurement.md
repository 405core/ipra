# 需求书

本文档对应《智能旅客风险评估与辅助问询系统采购需求书》，已转换整理为 Markdown，以便在当前知识库中进行统一浏览、搜索与后续维护。

## 一、项目概述

本项目旨在采购一套基于人工智能大模型和多模态交互技术的“智能旅客风险评估与辅助问询系统”。该系统需融合旅客背景数据、实时音视频采集、微表情与视线追踪及情感计算技术，通过大语言模型自动生成针对性问询策略，辅助一线检查员识别旅客是否存在隐瞒、虚假陈述或涉嫌出境犯罪（如跨境赌博等）的真实目的，提升口岸、海关、边检等场景的风险人员拦截效率与决策准确度。

## 二、业务功能需求

### 2.1 旅客基础画像管理模块

系统需具备强大的基础信息获取与整合能力，为后续的大模型推理提供准确的上下文基座。根据业务现状，所有旅客基础画像数据均来源于本地文件导入。

#### 2.1.1 画像数据离线导入

系统必须支持检查人员通过上传由采购方提供的 Excel 或 CSV 格式文件，批量导入旅客基础画像信息以及高风险人员名单。

#### 2.1.2 全息画像维度要求

系统需能够精准解析导入的表格文件，读取的画像数据维度必须涵盖个人基本信息、订票与行程信息、历史出行记录、职业背景以及违法犯罪记录等。

#### 2.1.3 证件信息快速匹配

数据导入完成后，系统应支持通过扫描或手动输入旅客证件号，直接从上述导入的表格数据中快速检索、匹配并全息展示待问询旅客的画像信息。如该旅客在表格中被标记为高风险人员，系统需进行显著的风险预警提示。

### 2.2 多模态实时感知模块

系统需具备现场音视频数据的实时采集与解析能力。

#### 2.2.1 视频流特征采集

系统需接入现场摄像头视频流，实时捕获并分析旅客的以下特征：

- 外貌体态特征
- 面部微表情变化
- 眼神动作与视线落点

#### 2.2.2 语音采集与实时转写（ASR）

系统需接入麦克风阵列，采集旅客的对话语音。要求系统具备高准确率的语音识别（ASR）能力，目前主要支持普通话，需将旅客的语音内容实时、精准地转录为文本格式，以便输入至大模型进行交互分析。

### 2.3 大模型智能辅助问询模块（核心）

系统需内置或集成大语言模型，并结合多模态数据实现动态、多轮的问询策略辅助。

#### 2.3.1 预评估与首轮策略生成

在获取旅客基础画像后，系统大模型需自动进行逻辑研判，并生成首轮问询问题清单。此轮目标为初步试探其出境真实目的，设计能暴露出前后逻辑不一致或隐瞒真实意图（如出境赌博）的诱导性问题。

#### 2.3.2 音视频融合情感研判与异常提示

针对问询中的旅客，情感计算需要将摄像头捕捉到的视频画面与旅客回答的语音内容深度结合，进行多模态综合判断。系统前端需具备直观的可视化展示能力，能够精准捕获并高亮显示异常瞬间，例如当旅客在回答特定问题或说到某个关键敏感词时，情绪上产生的明显变化，或眼神出现的躲闪等。结合基础画像、首轮答复及多模态情感分析结果，大模型需自动生成第二轮追问指引和提示话术，供工作人员参考。

#### 2.3.3 自适应多轮交互

系统需支持多轮次问询闭环。大模型应基于所有历史问询记录、前期基础信息以及实时多模态反馈，持续生成下一轮的问询提示与指引。

#### 2.3.4 交互轮次熔断控制

为保证现场通行效率与系统资源，系统必须支持管理员自定义“总交互轮次”上限阈值，达到设定轮次后系统应自动提示结束当前辅助流程。

### 2.4 人机协同与结论定性模块

系统应提供完善的问询结论研判与归档流程。

#### 2.4.1 人工辅助判定

整体问询结束后，系统需提供标准化选项供工作人员进行定性判断，结论选项至少应包含“隐瞒”“虚假陈述”“无异常”。

#### 2.4.2 结构化判定依据记录

系统强制要求检查员在做出最终判定后，必须输入做出该判断的详细理由，与完整问询情况一并归档，形成业务闭环。

## 三、非功能需求与安全审计约束

鉴于本系统涉及极其敏感的个人隐私、出入境数据及执法过程，系统必须满足以下严格的安全与审计要求：

### 3.1 全链路操作留痕（审计日志）

系统的所有交互行为，包括但不限于用户登录、数据查询、操作执行、日志查看等，必须 100% 自动记录并长期留存，形成不可篡改的操作留痕记录，满足事后审计追溯要求。

### 3.2 严密的防数据泄露控制（DLP）

系统需从底层架构实现“数据不落地、不外流”。禁止任何形式的系统数据导出，例如禁用导出按钮、禁用复制粘贴等，保证系统数据仅在受控环境内流转。

### 3.3 强制屏幕与输出水印

系统的所有操作前端页面、视频回放界面以及任何形式的输出内容，均必须强制叠加系统自带的数字水印，例如包含当前操作员警号或工号、时间戳等信息，以防止拍照录像泄密并实现泄密溯源。

### 3.4 软硬件运行环境与部署要求

- 后端服务需具备国产化适配能力，支持部署在昇腾（Ascend）或鲲鹏（Kunpeng）等国产算力服务器上。
- 前端服务需兼容并支持运行在 Windows 7 或 Windows 10 操作系统中。前端界面需兼容 13.5 英寸及以下尺寸的屏幕，且 UI/UX 必须针对触摸屏操作进行适配和优化。

### 3.5 数据提供与来源声明（新增）

本项目涉及的所有数据，包含但不限于 2.1 节中所述的旅客基础画像管理信息、高风险人员名单等业务数据，以及用于训练、情感计算的数据集，均由采购方统一提供。供应商或实施方需基于采购方提供的脱敏或受控数据进行系统的适配、部署与模型调优。
