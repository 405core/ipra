## 1. 数据出口清单与策略基线

- [ ] 1.1 盘点 `apps/backend/main.go` 和各个 handler 注册出的所有后端到前端数据出口，包括 JSON 接口、protected 接口、模板下载、媒体流、OCR 响应、导入响应和旧 inquiry 路由
- [ ] 1.2 将每个数据出口归类为强制明文例外、最小控制元数据、受保护业务内容或禁止明文交付，并记录每条路由的目标契约
- [ ] 1.3 盘点 `apps/frontend/src/app` 中所有数据消费模块，以及用户端和管理端视图中的所有页面级敏感状态来源
- [ ] 1.4 标注每个前端状态字段属于可接受的临时输入、可接受的最小控制元数据，还是被禁止的长生命周期敏感明文

## 2. 后端敏感数据出口收口

- [ ] 2.1 替换旅客检索展示所依赖的明文契约，使用户端不再依赖 `/api/passenger-profiles` 明文返回敏感展示内容
- [ ] 2.2 替换管理端基础画像、高风险名单、用户和审计展示所依赖的明文契约，使管理端列表不再依赖 `/api/admin/profiles`、`/api/admin/watchlist`、`/api/admin/users` 或 `/api/audit-logs` 的明文内容
- [ ] 2.3 为导入结果增加受保护明细交付机制，使行级 `errorDetails` 和 `rawData` 不再作为明文浏览器响应返回
- [ ] 2.4 为 OCR 结果增加受保护处理路径，使识别出的身份字段不再作为通用明文展示载荷返回
- [ ] 2.5 为迁移期仍暂时保留的明文旧路由增加显式弃用标记或访问约束，避免继续被受保护页面依赖
- [ ] 2.6 增加回归测试：一旦受保护浏览器展示路由暴露被禁止的敏感明文字段或存储定位信息，测试必须失败

## 3. 受保护辅助问询运行态重构

- [ ] 3.1 去掉受保护问询页面对明文 `searchPassengerProfiles()` 的依赖，并替换为后端驱动的受保护会话初始化路径
- [ ] 3.2 去掉受保护问询页面为了展示而对明文 `fetchMemoryContext()` 的依赖，将记忆展示完全切换到受保护记忆资产
- [ ] 3.3 重构受保护问询的追问、摘要和辅助判断流程，使浏览器只发送推进服务器会话所需的最小控制载荷
- [ ] 3.4 在存在受保护替代方案后，移除前端中旅客画像、记忆上下文、judgement briefing、归档快照等长生命周期明文副本
- [ ] 3.5 将 `UserAskView` 中明文转写、由 ASR 派生的回答文本和轮次摘要展示替换为受保护运行态展示
- [ ] 3.6 审查旧的 plain inquiry 接口，移除 `/api/inquiry/sessions*` 和 `/api/asr/transcribe` 对敏感展示行为的浏览器依赖

## 4. 受保护归档与媒体交付

- [ ] 4.1 将 `ManagementView` 中归档列表的明文展示替换为受保护归档行资产，覆盖旅客身份、操作者身份、判定和摘要字段
- [ ] 4.2 将 `ManagementView` 中归档详情的明文展示替换为受保护资产，覆盖判定理由、旅客快照、问题内容、转写内容、轮次摘要及相关文本块
- [ ] 4.3 从面向浏览器的归档元数据契约中移除 `videoUrl`、`minioBucket`、`minioObjectKey` 等存储定位字段
- [ ] 4.4 引入受保护归档播放句柄或等价的后端控制播放元数据，在保留流式播放能力的同时避免泄露存储定位信息
- [ ] 4.5 在收紧元数据契约后，确保归档视频播放仍持续带水印、受访问控制并可审计

## 5. 管理端与用户端页面状态最小化

- [ ] 5.1 重构 `ManagementView` 的列表筛选和选择逻辑，使受保护展示页不再需要完整明文数组来生成本地搜索文本、筛选选项或详情重建数据
- [ ] 5.2 引入显式的单记录编辑加载路径，使管理端编辑弹窗只请求当前表单所需的最小明文字段
- [ ] 5.3 重构 `UserHomeView`、`UserAuditView` 及相关布局流程，确保导入状态、检索状态和受保护详情弹窗不再把行级敏感明细作为明文 UI 状态泄露
- [ ] 5.4 复核由会话派生的身份展示，确认前端状态中只保留导航和水印上下文所必需的最小操作者身份信息

## 6. 验证、审计与最终收口

- [ ] 6.1 增加自动化后端和前端检查，扫描受保护路由和受保护页面中是否仍存在旅客、问询、归档、审计、OCR、导入或存储路径相关的被禁止明文字段
- [ ] 6.2 按采购需求书重新做逐路由 DLP 合规复核，确认每个后端数据出口都有明确分级和落地结果
- [ ] 6.3 验证强制明文例外是否严格限制在登录、当前用户会话、非敏感设置、空白模板下载和用户主动输入数据这些范围内
- [ ] 6.4 形成最终的“严格去明文收口”核查报告，覆盖后端路由、前端服务、页面状态、媒体交付和残余允许例外
